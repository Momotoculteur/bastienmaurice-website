## Un example d'avant - Une application JAVA

Les dév font du logiciel

Envoi des fichiers .WAR sur un serveur FTP
Envoie un fichier de properties en format Excel (Creds, Script d'install avec des tokens/clée/Etc, paramètrer certains hyper-paramètres)

Les Ops récupèrent et pousse le logiciel sur le serveur en remplaçant par les bonnes values

## C'est quoi DevOps ?

- Combinaison de philos et de pratiques
- But d'améliorer la capacité d'une entreprise à livrer
- Tenir un rythme élevé
- Mieux servir le client, et donc gain en compétitivité



![devops-model](./img/devops-model.png)

---


## Fonctionnement du DevOps

On rassemble les équipes de développement, d'opérations, QA...

Travaille sur l'ensemble du cycle de vie de l'application :

- Développement
- Tests
- Deploiement



---


## Avantages du DevOps

- Meilleure cohésion entre les équipes, meilleur comunication
- Amène de melleurs feedback, plus rapide, moins de friction
- Permet d'avoir des cycles, implémentation de feature plus rapide
- Meilleur qualité & sécurité du code
